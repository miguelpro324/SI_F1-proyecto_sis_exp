WHAT WAS BROKEN & HOW IT'S FIXED
================================================================================

PROBLEM 1: Type Conversion
  Issue: (read) returns atoms, not FLOAT/INTEGER values
  Fix:   Added explicit conversion
    (bind ?fpg (float (read)))  ← WRONG (doesn't convert)
    (bind ?fpg-raw (read))
    (bind ?fpg (float ?fpg-raw))  ← RIGHT (converts properly)

PROBLEM 2: Duplicate Output
  Issue: Rules fired multiple times for same pattern
  Fix:   Added guard facts to prevent re-firing
    (defrule output-diagnosis-display
      ...
      (not (shown-diagnosis))  ← Only fire if not yet shown
      =>
      (assert (shown-diagnosis))  ← Prevent future firing
      ...)

PROBLEM 3: Incomplete Error Handling
  Issue: No guard against repeat error messages
  Fix:   Same guard mechanism applied to all output rules

================================================================================
KEY FIXES IN DETAIL
================================================================================

FIX 1: Type Conversion

  WRONG:
    (bind ?fpg (read))
    (assert (patient (fpg ?fpg)))
    → ?fpg is an atom (symbol), not FLOAT
    → Template validation fails

  CORRECT:
    (bind ?fpg-raw (read))
    (bind ?fpg (float ?fpg-raw))
    (assert (patient (fpg ?fpg)))
    → ?fpg is properly converted to FLOAT
    → Template validation succeeds

FIX 2: Guard Facts

  WRONG:
    (defrule output-rule
      (fact1)
      (fact2)
      =>
      (printout t "Output" crlf))
    → Fires every time both facts exist
    → Output appears multiple times

  CORRECT:
    (defrule output-rule
      (fact1)
      (fact2)
      (not (rule-fired))
      =>
      (assert (rule-fired))
      (printout t "Output" crlf))
    → Guard fact (rule-fired) prevents re-firing
    → Output appears exactly once

FIX 3: Error Handling

  WRONG:
    (defrule output-error
      (system-state (phase error))
      =>
      (printout t "Error" crlf))
    → Fires repeatedly whenever error phase active

  CORRECT:
    (defrule output-error
      (system-state (phase error))
      (not (shown-error))
      =>
      (assert (shown-error))
      (printout t "Error" crlf))
    → Guard prevents repeats

================================================================================
TROUBLESHOOTING - COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE: "Invalid integer" or "Invalid float" error
  CAUSE: Non-numeric input
  SOLUTION: Enter numbers only (1001, 145.0, 7.2)

ISSUE: "Undefined function integer" or "Undefined function float"
  CAUSE: Old CLIPS version or incompatible
  SOLUTION: Ensure CLIPS 6.30+
  FIX: Use (int ?x) instead if needed

ISSUE: No output after running
  CAUSE: File not loaded or syntax error
  SOLUTION: 
    1. Check: (rules) to see loaded rules
    2. Verify all 5 files loaded
    3. Check syntax: (batch knowledge_base/04_cli.clp)

ISSUE: Duplicate output appearing
  CAUSE: This version shouldn't have this
  SOLUTION: Verify you're using FIXED version (04_cli.clp dated 2026-04-10)

ISSUE: CLIPS console hangs at (read)
  CAUSE: Waiting for user input
  SOLUTION: Type value and press Enter

ISSUE: "Unknown variable" error
  CAUSE: File not loaded (templates not defined)
  SOLUTION: Make sure 00_templates.clp loaded first
