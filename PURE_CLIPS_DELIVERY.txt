================================================================================
INPUT CAPTURE MECHANISM
================================================================================

CLIPS (read) Function:
  (bind ?patient-id (read))
  (bind ?fpg (read))
  (bind ?hba1c (read))

Standard Input:
  - (read) captures user input from stdin
  - User presses Enter after each value
  - Values converted to appropriate types (INTEGER, FLOAT)
  - Type validation handled by CLIPS template definitions

Fact Assertion:
  (assert (patient (id ?patient-id) (fpg ?fpg) (hba1c ?hba1c)))
  (assert (system-state (phase validation)))

ARCHITECTURE
================================================================================

Pure CLIPS Stack:

  User Interface Layer (04_cli.clp)
    ├─ Input: (read) from user
    └─ Output: (printout) to console

  Inference Engine Layer (CLIPS runtime)
    ├─ Pattern matching
    ├─ Conflict resolution
    └─ Rule firing

  Knowledge Base Layer (00-03.clp)
    ├─ Fact templates
    ├─ Validation rules
    ├─ Abstraction rules
    └─ Diagnostic rules

  Execution:
    (initial-fact) → (run) → Rules fire in sequence → Diagnosis output
